PHP Sürüm Karşılaştırması Yapmak

is_phpversion( string $surum_bilgisi = '5.2.4' )

Parametre olarak yazılan sürümün desteklenip desteklenmediğini bilgisini true veya false çıktısı ile belirtir.

NoParametrelerAnlamları
1StringDesteklenme durumu öğrenilecek sürüm bilgisi.

echo is_phpversion('5.6.0'); // Çıktı: false.

Değişkenlerin Tanımlanma Durumunu Kontrol Etmek

is_defined( variable $degisken )

Bu yöntem, değişkenlerin tanımlanıp tanımlanmadığını belirlemek için kullanılır. Değişken tanımlı olduğunda true yani 1 değeri döner. Eğer değişken 0 da dahil herhangi bir değer almamışsa false değeri döner.

NoParametrelerAnlamları
1DeğişkenTanımlanıp tanımlanmadığı kontrol edilecek değişken.

$degisken = 0;
echo is_defined($degisken); // Çıktı: 1
$degisken = NULL;
echo is_defined($degisken); // Çıktı: 0

Değişkenlerin Tanımlanmama Durumunu Kontrol Etmek

undefined( variable $degisken )

Bu yöntem, is_defined() yöntemine benzemekle birlikte çıktı değerlerinde küçük bir farklılık vardır. Eğer değişken, NULL dışında herhangi bir değer almışsa false, almamışsa true değeri döner.

NoParametrelerAnlamları
1DeğişkenTanımlanıp tanımlanmadığı kontrol edilecek değişken.

$degisken = 0;
echo undefined($degisken); // Çıktı: 0
$degisken = NULL;
echo undefined($degisken); // Çıktı: 1

Bir Dosyanın Dahil Edilme Durumunu Kontrol Etmek

is_import( string $dosya_yolu )

Herhangi bir dosyanın, require veya include yöntemlerinden herhangi biriyle daha önce dahil edilip edilmediği bilgisini verir. Herhangi bir dosya daha önce dahil edilmişse true değeri, edilmemişse false değeri döner.

NoParametrelerAnlamları
1Dosya YoluDahil edilip edilmediği kontrol edilecek dosyanın yolu.

echo is_import('OrnekDosya.php'); // Çıktı: false veya true

Bir Dosyanın Var Olma Durmunu Kontrol Etmek

is_file_exists( string $dosya_yolu )

Herhangi bir dosyanın, var olup olmama durumunu kontrol etmek amacıyla kullanılır. Bu yöntemin is_file() ve file_exists() yöntemlerinden farkı, yönteme girilen parametrenin dosya yolu olup olmadığını aynı zamanda öyle bir dosyanın var olup olmadığının kontrolünü yapmasıdır. Yani is_file_exists(), bu tanımlı olan 2 yöntemin görevini aynı anda yapar. Ayrıca diğer tanımlı olan yöntemler, içerisinde URL adresi barındıran verilerde dosyanın var olup olmadığı hakkında bilgi vermezken, is_file_exists() yöntemi, bilgi verir.

NoParametrelerAnlamları
1string Dosya YoluKontrol edilecek dosyanın yolu.

echo is_file_exists('http://www.siteadi.xxx/Ornek/dosya.php'); // Çıktı: true
echo file_exists('http://www.siteadi.xxx/Ornek/dosya.php'); // Çıktı: false
echo is_file('http://www.siteadi.xxx/Ornek/dosya.php'); // Çıktı: false

Bir Dizinin Var Olma Durmunu Kontrol Etmek

is_dir_exists( string $dizin_yolu )

Herhangi bir dizinin, var olup olmama durumunu kontrol etmek amacıyla kullanılır. Bu yöntemin is_dir() yönteminden farkı, yönteme girilen parametrenin dizin yolu olup olmadığını aynı zamanda öyle bir dizinin var olup olmadığının kontrolünü yapmasıdır. Özellikle yol bilgileri, içerisinde URL adres bilgisi içeriyorsa tanımlı olan diğer 2 yöntem, dizinin var olup olmadığı konusunda bilgi vermezken, is_dir_exists() yöntemi bilgi verir.

NoParametrelerAnlamları
1string Dizin YoluKontrol edilecek dizinin yolu.

echo is_dir_exists('http://www.siteadi.xxx/Ornek/Deneme'); // Çıktı: true
echo file_exists('http://www.siteadi.xxx/Ornek/Deneme'); // Çıktı: false
echo is_dir('http://www.siteadi.xxx/Ornek/Deneme'); // Çıktı: false

Farklı İşletim Sistemlerine Göre Satır Sonu Karakterini Kullanmak

ln(numeric $tekrar_sayisi = 1 )

İşletim sistemlerindeki farklılıktan kaynaklanan \n, \r karakter kullanımlarından doğan problemleri kaldırmak için oluşturulmuştur.

NoParametrelerAnlamları
1numeric Tekrar SayısıKarakterlerin kaç kez tekrarlanacağı.

echo ln(1); // Çıktı: WIN \n\r , MAC \r , OTHER \n
echo ln(3); // Çıktı: WIN \n\r\n\r\n\r , MAC \r\r\r , OTHER \n\n\n

Sitenin Dil İçeriğini Ayarlamak

set_lang( [ string $dil = 'tr' ] )

Sitenin dil içeriğini ayarlamak için kullanılır. Parametrenin varsayılan değeri 'tr', yani türkçedir. Sistemde varsayılan olarak iki dil ayarı mevcuttur. Bunlar; tr ve en 'dir. Bu ayarları değiştirmek yada daha fazla dil seçeneği eklemek için gerekli ayarlamaları Config/Language.php dosyasından yapmanız gerekmektedir. Unutmamanız gereken nokta ne kadar çok dil seçeneği eklerseniz Language/ dizini içerisindeki dil dizinlerinin sayısını da o denli artırmanız gerekmektedir. Yani Language/ dizinini kontrol ettiğinizde varsayılan olarak 2 adet dizin olduğunu görebilirsiniz. Bunlar; turkish ve english dizinleridir. Dikkat ederseniz dil dosyasının ayarları ile dil dizini içerisindeki dizinler arasında paralellik mevcuttur.

NoParametrelerAnlamları
1string Dil = "tr"Varsayılan olarak tr ayarlıdır.

echo get_lang(); // Çıktı: tr
set_lang('en');
echo get_lang(); // Çıktı: en

Sitenin Dil İçeriğini Öğrenmek

get_lang() ve current_lang()

Sitenin ayarlanmış dilini öğrenmek için kullanılır. Bu işlemi gerçekleştirmek için iki yöntem vardır. Bunlardan birincisi get_lang() fonksiyonudur diğeri ise current_lang() fonksiyonudur. Esasen ikisinin de yaptığı iş aynıdır ancak aralarında bir fark vardır. Bu fark current_lang() yönteminin kullanılabilmesi için Config/Uri.php dosyasındaki $config['Uri']['lang'] = false; değerinin true olarak ayarlanması gerektiğidir. Bu ayar false olarak ayarlanırsa current_lang() yöntemi false değeri döndürür yani hiç bir çıktı vermez.

echo get_lang(); // Çıktı: tr
set_lang('en');
echo get_lang(); // Çıktı: en
echo current_lang(); // Çıktı: false

Bir Veriye Önek Eklemek

prefix( string $veri , [ string $onek = '/' ] )

Herhangi bir metinsel veriye ön ek eklemek için kullanılır.

NoParametrelerAnlamları
1string VeriÖn ek eklenecek veri.
2[ string Önek = "/" ]Ön ek.

echo prefix('Veri'); // Çıktı: /Veri
echo prefix('Veri', '+++'); // Çıktı: +++Veri

Eğer eklenecek ön ek, verinin önünde zaten varsa ayrıca ekleme işlemi yapmaz.

echo prefix('/Veri'); // Çıktı: /Veri
echo prefix('+++Veri', '+++'); // Çıktı: +++Veri

NoParametrelerAnlamları

2[ string Sonek = "/" ]Sonek.

echo suffix('Veri'); // Çıktı: Veri/

Eğer eklenecek son ek, verinin sonunda zaten varsa ayrıca ekleme işlemi yapmaz.

echo prefix('Veri/'); // Çıktı: Veri/
echo prefix('Veri+++', '+++'); // Çıktı: Veri+++

Sitenin URL Adresini Öğrenmek

site_url( [ string $url_eki ] , [ numeric $dizin_indeksi = 0 ] )

Sitenin URL adresini öğrenmek için kullanılır. Config/Uri.php dosyasından ayarlar yapılırsa bu değişikliklerde dikkate alınarak sitenin ayarlanmış URL adresini verir.

NoParametrelerAnlamları
1string URL EkiSite URL adresine ek olarak yazılacak URI ifadeleri.
2[ string Dizin Indeksi ]URL'nin üst dizinlerine ulaşabilmek için kullanılır.

echo site_url(); // Çıktı: http://www.siteadi.xxx/zntr/index.php/
echo site_url('ornek/uygulama'); // Çıktı: http://www.siteadi.xxx/zntr/index.php/ornek/uygulama/
echo site_url('ornek/uygulama', -1); // Çıktı:http://www.siteadi.xxx/index.php/ornek/

Eğer Config/Uri.php dosyasından ssl, dil ve index.php ayarlarıda yapılmış olursa bu değişikliklerde dikkate alınır.

echo site_url(); // Çıktı: https://www.siteadi.xxx/tr/zntr/
echo site_url('ornek/uygulama'); // Çıktı: Çıktı: https://www.siteadi.xxx/tr/zntr/ornek/uygulama

Sitenin Kök URL Adresini Öğrenmek

base_url( [ string $url_eki ] , [ numeric $dizin_indeksi = 0 ] )

Sitenin Kök URL adresini öğrenmek için kullanılır. Config/Uri.php dosyasındaki ayarlardan etkilenmez. Genellikle bu yöntem gerçek dosyaları kaynak olarak göstermek için kullanılır. Özellikle resim, ses veya vidyo dosyalarının yolu belirtilmek istendiğinde tercih edilir.

NoParametrelerAnlamları
1string URL EkiSite URL adresine ek olarak yazılacak URI ifadeleri.
2[ string Dizin Indeksi ]URL'nin üst dizinlerine ulaşabilmek için kullanılır.

echo base_url(); // Çıktı: http://www.siteadi.xxx/zntr/
echo base_url('ornek/uygulama/resim.jpg'); // Çıktı: Çıktı: http://www.siteadi.xxx/zntr/ornek/uygulama/resim.jpg
echo base_url('ornek/uygulama/resim.jpg', -1); // Çıktı: Çıktı: http://www.siteadi.xxx/ornek/uygulama/resim.jpg

Config/Uri.php dosyasında yer alan dil ve index.php ayarlarındaki değişiklik base_url() yönteminde herhangi bir değişikliğe neden olmaz.

echo base_url(); // Çıktı: https://www.siteadi.xxx/zntr/
echo base_url('ornek/uygulama/resim.jpg'); // Çıktı: Çıktı: https://www.siteadi.xxx/zntr/ornek/uygulama/resim.jpg

Yol Bilgilerine Ulaşmak

path_info( string $dosya_yolu , [ string $bilgi = 'basename' ] )

Yolu gösterilen dizin veya dosyaya ait çeşitli bilgilere ulaşmak için kullanılır.

NoParametrelerAnlamları
1string Dosya YoluDosyanın yolu.
2[ string Bilgi = "basename" ]Yol hakkında istenen bilgi türü.
2. Parametrenin Alabileceği Değerler
basenameDizin veya dosya yolunun son segmentini(bölümünü) verir.
dirnameYol bilgisine ait dizinin ismini verir.
filenameYol bilgisine ait dosyanın ismini verir.
extensionYol bilgisine ait dosyanın uzantısını verir.

echo path_info('ornek/uygulama/deneme.php'); // Çıktı: deneme.php
echo path_info('ornek/uygulama/deneme.php' , 'dirname'); // Çıktı: ornek/uygulama
echo path_info('ornek/uygulama/deneme.php' , 'filename'); // Çıktı: deneme
echo path_info('ornek/uygulama/deneme.php' , 'extension'); // Çıktı: php

Metinsel İfadeyi Bölmek ve İstenen Parçaya Ulaşmak

divide( string $metin , [ string $bolunen_ifade = '|' ] , [ string/int $parca_numarasi = 0 ] )

Parametre olarak girilen metin bilgisini, belirlenen karaktere göre parçalamak ve istenilen parçayı almak için kullanılır.

NoParametrelerAnlamları

3[ string/int Parça Numarası = 0 ]Parça numarası.
Parça Numarası Parametresinin Alabileceği Değerler
Sıfır veya Pozitif SayıBaştan parça numarası.
Negatif SayıSondan parça numarası.
lastSon parça numarası.
firstİlk parça numarası.

echo divide('bilgi|zntr|framework'); // Çıktı: bilgi
echo divide('bilgi-zntr-framework', '-', 1); // Çıktı: zntr
echo divide('bilgi-zntr-framework', '-', -1); // Çıktı: framework
echo divide('bilgi?zntr?framework', '?', 'last'); // Çıktı: framework
echo divide('bilgi+zntr+framework', '+', 'first'); // Çıktı: bilgi

Hızlı Kütüphane ve Yöntemlerini Kullanmak

library( string/array $sinif_dosyasi_adi , string $fonksiyon_adi , [ array $parametreler ] )

Bu yöntem kullanmak istediğiniz kütüphanenin yöntemlerine kolay erişim sağlamak için oluşturulmuştur. Genel olarak kütüphanelerin dosya isimleri ile sınıfı isimleri aynıdır ancak bazı kütüphanelerin sınıf ismi kısaltımıştır. Bunlardan birine örnek vermek gerekirse Database.php kütüphanesinin sınıf adı Db'dir. Bu gibi sınıfları bu yöntemle kullanmak isterseniz birinci parametresi olan sınıf ismine dizi bilgisi girmeniz gerekmektedir. Örnek vermek gerekirse array('Database' => 'Db') şeklinde kullanılmalıdır.

NoParametrelerAnlamları
1string/array Sınıf AdıÇağrılacak dosyanın sınıf yada dosya adı.
2string FonksiyonKütüphanenin kullanılacak olan yönteminin adı.
3[ array Parametreler ]Varsa yöntemin parametreleri.

echo library('Encode', 'golden', array('Metni Şifrele', 'Şifre Eki')); // Çıktı: bc35185a25c5aa7330114a28525bee76

Eğer kütüphanenin dosya adı ve sınıf adı birbirinden farklı ise şu şekilde yazıyoruz.

library(array('Cookie' => 'Cook'), 'insert', array('yeni_cerez', 'Yeni Değer'));
echo library(array('Cookie' => 'Cook'), 'select', array('yeni_cerez')); // Çıktı: Yeni Değer
